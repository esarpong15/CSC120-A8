- Which methods did you decide to `overload`, and why?

For the House Class, I overloaded the
    - Constructor to take in Name + Address
    I wanted there to be a default generated house incase I'd only intend to fill out the name and address fields. That way it'll take in those parameters and output a house with 3 floors, no dining, and no elevator.
    - the moveOut() method
    What if I wanted to be an evil landlord and kick everyone out. Now I have a method to do so which will clear the ArrayList completely mimicing the action of a full eviction.

For the Library Class, I overloaded the
    - Constructor to only account for a Name + Address
    I wanted a constructor that would only need a name and adress when being made with default values of 5 floors and that it has an elevator.
    - Constructor to only account for a Address
    I utilized the constructor before hand to use the same default of 5 floors and an elevator, this time only taking in a parameter of an address, since the name parameter will just be filled in with '<Unknown Name>'.

For the Cafe Class, I overloaded the 
    - The sellCoffee() method
    To instead of just taking in int type values from the initial method to also consider double type values, but keeping the amount of cups to still be an int, since a phyisical cup realistically can't be a fraction of an amount.
    - The restock() method
    If i'm going to change the selling amount of stuff, I gotta make sure the restock can also reflect this too.
